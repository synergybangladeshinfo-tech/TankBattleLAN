# Tank Battle LAN

Offline local-multiplayer tank arena for Android. Up to **4 players**, each on their own phone, over **Wi-Fi hotspot or any shared LAN** — no internet, no ads, no login.

- Unity **6000.0 LTS** (any 6000.0.x patch)
- **Netcode for GameObjects 2.4** for game traffic + custom **UDP broadcast** for automatic host discovery
- Third-person low-poly 3D, virtual joystick + fire button, health/respawn, kill scoreboard, 5-minute matches with a win screen, 5 maps, pause menu, sound settings and 3 graphics-quality levels, 60 FPS target

## Requirements

- Unity Hub with **Unity 6000.0 LTS** and the **Android Build Support** module (includes OpenJDK, SDK & NDK — tick all three when installing)
- Android phones running Android 6.0+ (API 23)

## Build instructions

1. **Open the project**: Unity Hub → *Add* → select the `TankBattleLAN` folder → open with 6000.0 LTS. First import takes a few minutes (packages download from the Unity registry; this is the only step that needs internet — builds and gameplay are fully offline).
2. **Generate the content**: menu bar → **Tank Battle → 1. Generate Everything**. This creates all materials, the `Tank` / `Bullet` / `NetworkManager` prefabs, the `MainMenu` scene, the five map scenes, registers everything in Build Settings and applies the Android player settings (IL2CPP, ARM64+ARMv7, landscape, INTERNET permission only). It is idempotent — rerun any time to regenerate.
3. **Build the APK**: **Tank Battle → 2. Build Android APK** (switches build target if needed). Output: `Builds/TankBattleLAN.apk`. Alternatively use *File → Build Profiles → Android → Build* yourself.
4. **Install** the same APK on every phone (`adb install TankBattleLAN.apk` or copy the file over).

### Quick test in the editor

After step 2 the `MainMenu` scene is open — press Play, tap **HOST GAME**, pick a map and **START MATCH**. WASD/arrows drive, Space fires (editor-only fallback). For a two-instance LAN test on one PC, make a desktop build and run it next to the editor, or use a clone tool such as ParrelSync.

## How to play

1. One player turns on a **Wi-Fi hotspot** (or everyone joins the same Wi-Fi network) and taps **HOST GAME** → picks one of the 5 maps → **START HOSTING**.
2. Everyone else taps **JOIN GAME** — hosts on the network appear automatically within a second or two; tap one to join. No IP addresses to type.
3. The host presses **START MATCH**. 5 minutes, most kills wins. Destroyed tanks respawn after 3 s. Late joiners drop straight into the running match (up to 4 players total).
4. After the win screen the host can pull everyone **back to the lobby** for a rematch.

## Architecture notes

- **Host = server + player.** Tanks are owner-authoritative (`ClientNetworkTransform`) for lag-free touch controls; everything that matters (bullets, damage, kills, timer, respawns) is **server-authoritative**.
- **Discovery**: clients broadcast a probe on UDP `47777` (global + per-interface directed broadcast); the host replies with name/map/player-count; game traffic then runs over Unity Transport on UDP `7777`.
- **Connection approval** enforces the 4-player cap and carries each player's name as the connect payload.
- **Scenes/prefabs are generated by editor scripts** rather than shipped as hand-written YAML — this guarantees clean serialization on any 6000.0.x version. UI is built at runtime by `UIFactory` so every reference is wired by construction.
- **Audio is synthesized at startup** (chiptune loops + SFX) so the repo ships zero binary assets. Swap in real clips by assigning fields in `AudioManager`.
- **Performance**: static-batched primitive geometry, one directional light, solid-color camera clear, no fog/skybox/bake, 30 Hz net tick, `targetFrameRate = 60`, quality presets that disable shadows/AA on Low.

## Permissions & privacy

The manifest ends up with **INTERNET only** — Android requires it for *any* socket, including pure-LAN UDP; the game never contacts the internet. No other permissions, no ads, no login, no analytics.

## Troubleshooting

- **No hosts found**: all phones must be on the *same* network (the hotspot owner should host). Some routers block UDP broadcast between clients ("AP/client isolation") — use a phone hotspot instead. A few Android devices throttle broadcast *reception*; joining from those devices works best when the other phone hosts.
- **Port in use** when hosting: another app is holding UDP 7777/47777 — reboot or change the ports in `GameConstants`.
- **Build errors about Android SDK/NDK**: ensure the Android module (with SDK/NDK/OpenJDK) is installed for the exact editor version via Unity Hub.
